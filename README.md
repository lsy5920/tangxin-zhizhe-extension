# 糖心志者

糖心志者是一个 Chrome Manifest V3 浏览器插件，用于账号池管理、播放状态展示、视频下载、版本更新和缓存清理。

## 项目介绍

当前界面不是手写相似样式，而是直接把 Figma Make「生成糖果风页面」的 React 页面源码接入插件工程：源码放在 `ui-src/`，通过 Vite 打包为 `dist-ui/txzz-ui.js` 和 `dist-ui/txzz-ui.css`，再由 `manifest.json` 作为内容脚本加载。`content.js` 只保留账号池、播放详情、下载和更新等业务能力，并通过事件状态桥把真实流程交给 React 界面展示；旧 DOM 面板、旧流程浮层和旧更新弹窗已经删除。

插件默认只显示左上角流程标签和右下角「志」字糖果悬浮球，不再用整屏背景遮挡网站内容。点击悬浮球后展开粉紫糖果风控制台，打开面板时会显示模糊蒙版，桌面端使用左侧导航和右侧工作区，移动端使用全屏面板和底部导航，把账号池、播放状态、下载任务、版本更新和缓存清理集中管理。

## 环境要求

- 操作系统：Windows 10 或 Windows 11
- 浏览器：Chrome `120.0` 及以上，或 Edge `120.0` 及以上
- 插件规范：Chrome Manifest V3
- 本地检查工具：Node.js `22.16.0` 及以上
- 本机验证版本：Node.js `25.8.0`，npm `11.11.0`
- 前端界面：React `18.3.1`，React DOM `18.3.1`，lucide-react `0.487.0`
- 构建工具：Vite `8.0.16`，@vitejs/plugin-react `6.0.2`，Tailwind CSS `4.1.12`，@tailwindcss/vite `4.3.1`
- 远程服务：Cloudflare Worker
- 云端数据库：Supabase
- 部署工具：Wrangler `4.98.0`
- 视频转封装：`mux.js@7.0.0`
- 文件编码：UTF-8

## 核心功能

- 悬浮球入口：默认只显示「志」字悬浮球和左上角流程标签，支持桌面端和移动端拖动，不遮挡网站主体内容。
- 糖果风控制台：根据 Figma Make 设计稿重做粉紫渐变、白色卡片、圆润按钮、轻量阴影和糖果悬浮球，并补充 Shadow DOM 圆角变量兜底，避免按钮和色块变直角。
- 新界面强制接管：旧 DOM 面板、旧流程标签和旧更新弹窗已删除，浏览器页面上唯一可见入口为 Figma Make React 糖果风界面。
- 图片素材美化：内置品牌控制台插画、空状态插画和功能图标素材，配合糖果风主题提升总览页、导航和空状态视觉质感。
- 五页控制台：导航精简为总览、账号池、播放、下载、设置，去除普通用户不需要的调试入口。
- 总览页面：集中显示当前展示状态、账号池数量、播放记录数量、下载任务和最近流程。
- 页面状态展示：集中展示会员状态、扩展运行状态、账号池状态和最近流程。
- 播放状态展示：展示最近视频、播放状态、分片统计和下载入口，不再暴露原始接口堆栈。
- 账号池管理：账号池页重构为控制台布局，支持本地账号、云端自动轮换和云端优先本地兜底。
- 云端轮换策略：云端账号按金币数量升序排序；获取播放详情失败或金币视频处理失败时自动切换下一个云端账号。
- 金币视频策略：先轮询所有云端账号是否已购买；如果全部未购买，则从金币数量最少的账号组中随机选择一个账号购买，失败后继续按同规则轮换。
- 云端账号卡片：统一展示账号昵称、可用状态、不可用原因、普通 VIP、尤物圈和金币余额；云端账号不提供手动选择按钮，只参与自动轮换；失效云端账号默认隐藏，可手动查看。
- 云端分类识别：前端账号池页面和后台统一识别 `cloudReadonly`、`isCloud`、`remoteId`、`cloudId`、`source=remote`、`source=qrcode`、`source=remote-seed`、`source=seed` 等云端摘要标记，避免云端账号误显示到本地账号分类。
- 账号可用判定：账号池会区分账号凭据错误和视频资源异常，视频下架、播放详情缺少链接或临时接口异常不会把有成功检查记录的云端账号误判为失效。
- 关键流程提示：插件运行时会在页面左上角显示轻量流程标签，只展示同步账号池、云端账号轮换、当前使用账号、金币解锁、获取播放状态、下载合并、保存到设备和异常等关键状态。
- 添加账号面板：通过「添加账号」按钮展开，支持账号密码、账号凭证、token/deviceId 三种本地保存方式。
- 远程同步：通过 Cloudflare Worker 读取 Supabase 账号池，只在插件侧展示脱敏摘要。
- 播放页面：通过 Worker 获取播放详情，并在面板中展示用户可读的播放状态和下载入口。
- 下载管理：接管视频详情页下载按钮，创建后台下载任务。
- MP4 输出：M3U8 分片下载完成后优先转封装为真实 MP4，失败时保留 TS 兜底。
- 数据清理：支持一键清除旧版本缓存、账号池缓存、播放缓存和页面运行缓存。
- 更新提醒：优先读取远程仓库 `update.json` 版本清单，对比版本号、构建号和最新日志 ID；读取失败时自动降级为 README 更新日志对比；设置页提供「检查更新」按钮。
- 移动端适配：手机视图下顶部操作栏和底部导航固定，内部内容独立滚动，并使用更接近设计稿的全屏糖果风面板。

## 安装教程

1. 打开 Chrome 或 Edge。
2. 在地址栏输入 `chrome://extensions/` 并回车。
3. 打开右上角「开发者模式」。
4. 点击「加载已解压的扩展程序」。
5. 选择当前目录：`tangxin-zhizhe-extension`。
6. 如果刚修改过 Figma Make 界面源码，先在当前目录执行 `npm install`，再执行 `npm run build` 生成 `dist-ui/`。
7. 初次只使用已构建好的插件时，可以直接加载当前目录；如浏览器提示缺少 `dist-ui/txzz-ui.js`，说明需要先执行上一步构建命令。
8. 打开需要联调的页面。
9. 页面右下角出现「志」字悬浮球，表示插件加载成功。

## 远程账号池配置

推荐使用远程账号池保存账号凭据。账号密码、二维码凭证、token、deviceId、Supabase `service_role` 等敏感信息应放在 Cloudflare Worker 或 GitHub Secrets 中，插件前端只保存 Worker 地址。

配置步骤：

1. 点击「志」字悬浮球打开面板。
2. 进入「账号池」页。
3. 在「远程配置」中填写 `Worker URL`。
4. 选择账号来源模式。
5. 点击「保存配置」。
6. 点击「同步云端」读取云端账号池。
7. 云端账号默认只显示可用账号；需要排查坏账号时，打开「查看已失效云端账号」。

账号来源模式说明：

| 模式 | 说明 |
| --- | --- |
| 云端自动轮换 | Worker 按金币数量从少到多选择可用账号，失败时自动切换下一个账号。 |
| 本地选中账号 | 只使用插件本地保存的账号。 |
| 云端优先，本地兜底 | 优先使用云端账号，失败后尝试本地账号。 |

## 使用教程

### 打开和关闭面板

1. 打开目标页面。
2. 点击「志」字悬浮球。
3. 悬浮球隐藏，控制台展开，默认进入「总览」页。
4. 点击关闭按钮后，面板关闭，悬浮球重新出现。
5. 按住「志」字悬浮球拖动，可调整悬浮球位置。
6. 点击主界面右上角「关于」按钮，可打开项目主页。
7. 桌面端通过左侧导航切换页面，手机端通过底部导航切换页面。

### 应用页面状态展示

1. 打开插件面板。
2. 点击「应用展示覆盖」。
3. 面板会刷新当前状态区域。
4. 如果页面内有对应状态展示，插件会同步进行页面侧显示更新。

### 查看播放状态

1. 打开任意视频详情页或播放页。
2. 插件会自动记录最近视频和播放处理状态。
3. 进入「播放」页查看账号摘要、视频编号、分片统计和下载入口。
4. 如果当前视频支持下载，可直接点击「下载当前视频」创建下载任务。

### 管理账号池

1. 进入「账号池」页。
2. 顶部统计区会显示全部账号、云端可用账号、本地账号和失效账号数量。
3. 云端账号卡片统一显示账号昵称、可用状态、不可用原因、普通 VIP、尤物圈和金币余额。
4. 失效云端账号默认隐藏；打开「查看已失效云端账号」后可以查看不可用账号和失败原因。
5. 点击「添加账号」会先弹出类型选择窗口。
6. 选择「账号密码」后，只显示账号昵称、用户名和密码等必填内容。
7. 选择「账号凭证」后，只显示账号昵称和账号凭证内容。
8. 选择「token/deviceId」后，只显示账号昵称、deviceId 和 userToken。
9. 点击「保存本地」只保存到插件本地账号池。
10. 点击「保存并上传云端」会把账号提交给 Worker 加密保存。
11. 上传成功后，同步回来的云端卡片会继续显示账号昵称、凭据类型、可用状态、不可用原因、普通 VIP、尤物圈和金币余额。
12. 本地账号卡片可选择、编辑、检查、上传云端或移除；云端账号只显示摘要和「自动轮换」状态，不能手动固定选择，也不能在插件前端修改凭证明文。

### 下载视频

1. 打开视频详情页。
2. 点击页面上的下载或缓存按钮。
3. 插件会创建后台下载任务。
4. 如果是 MP4 直链，会直接进入可保存状态。
5. 如果是 M3U8，会先下载分片，再合并并优先转封装为 MP4。
6. 进入「下载」页查看进度。
7. 合并完成后点击「保存到设备」。

### 管理下载任务

「下载」页支持：

- 查看总任务、进行中、已完成和失败数量。
- 查看视频名称、视频编号、输出格式、文件大小和更新时间。
- 保存单个任务到设备。
- 删除单个任务卡片和缓存。
- 打开浏览器下载目录。
- 清空当前任务记录。

## 视频格式说明

插件处理 M3U8 时不会只改文件后缀，而是使用 `mux.js@7.0.0` 对 TS 分片合并结果进行转封装：

- 转封装成功：输出 `.mp4`，MIME 为 `video/mp4`。
- 转封装失败：保留 `.ts` 兜底文件，并在任务卡片显示失败原因。

## 本地账号上传云端

1. 进入「账号池」页。
2. 填写本地账号信息。
3. 点击「保存并上传云端」，或在本地账号卡片上点击「上传云端」。
4. 上传成功后，插件会同步云端账号池。
5. 已上传账号会转为云端摘要展示，不能在插件前端查看或修改凭证明文。

## 清除旧数据缓存

如果覆盖安装后出现旧账号、旧播放详情或旧配置残留：

1. 打开插件面板。
2. 进入「设置」页。
3. 点击「清除数据缓存」。
4. 确认后刷新页面。

清理范围包括：

- 插件本地账号池缓存
- 远程账号池摘要缓存
- 播放状态缓存
- 下载任务缓存
- 页面监听运行缓存
- 旧版本默认配置

## 常见问题排查

### 页面没有出现悬浮球

1. 确认加载的是 `tangxin-zhizhe-extension` 目录。
2. 确认浏览器扩展页没有报错。
3. 刷新页面。
4. 检查当前页面地址是否匹配 `manifest.json` 的 `matches` 配置。

### 手机端点击按钮无效

1. 刷新页面后重试。
2. 确认没有旧版本插件同时启用。
3. 检查页面是否被其他浮层遮挡。
4. 尝试关闭面板后重新点击悬浮球展开。

### 刷新后仍然看到旧页面

1. 在浏览器扩展管理页点击当前插件的「重新加载」按钮。
2. 确认加载目录中存在 `dist-ui/txzz-ui.js` 和 `dist-ui/txzz-ui.css`。
3. 重新打开或刷新目标页面，不要只刷新扩展管理页。
4. 当前版本已经删除旧 DOM 面板、旧流程浮层和旧更新弹窗；如果仍看到旧面板，通常是浏览器还在运行旧扩展实例，关闭重复安装的旧版本后再刷新页面。

### 远程账号池同步失败

1. 检查 Worker 地址是否可访问。
2. 检查 Worker 是否配置了 `SUPABASE_URL` 和 `SUPABASE_SERVICE_ROLE_KEY`。
3. 检查 Supabase 是否已执行 `schema.sql`。
4. 访问 `/v1/health` 查看必填变量是否齐全。

### 本地账号上传云端后显示无凭据

1. 确认插件版本为 `1.3.0` 或更高版本。
2. 当前版本已保留云端摘要中的 `hasPassword`、`hasQrcode`、`hasToken` 标记。
3. 如果旧云端账号仍显示无凭据，重新上传该账号或在 Worker 部署新版后重新同步云端账号池。
4. 云端账号如果确实没有任何凭据，会显示「不可用：账号没有可用凭据」。

### 保存账号后页面刷新

1. 确认插件版本为 `1.3.0` 或更高版本。
2. 当前版本已将添加账号弹窗中的按钮全部改为普通按钮，并拦截表单默认提交。
3. 如果仍然刷新，请检查是否同时启用了旧版本插件。
4. 关闭旧版本插件后刷新页面，再重新打开「账号池」页保存。

### 播放详情为空

1. 确认账号池中至少有一个可用账号。
2. 同步远程账号池后再试。
3. 确认云端账号中至少有一个可用账号。
4. 查看 Worker 日志确认服务端接口是否返回错误。

### 下载按钮没有反应

1. 进入「下载」页点击「刷新下载」。
2. 确认视频详情页上的下载按钮已经渲染完成。
3. 检查浏览器是否拦截下载。
4. 如果任务正在下载 M3U8 分片，请等待显示「合并完成，待保存」。

### 保存到设备没有弹窗

1. 检查浏览器默认下载目录是否可写。
2. 查看浏览器下载记录。
3. 如果希望每次选择位置，请在浏览器下载设置中开启「下载前询问保存位置」。
4. 如果插件显示 TS 兜底，说明 MP4 转封装失败但文件仍可保存。

### 检查更新没有弹窗

1. 确认当前安装的插件版本已经包含更新检查功能，旧到完全没有该功能的版本无法自动提醒。
2. 确认远程仓库默认分支存在 `update.json`，地址为 `https://raw.githubusercontent.com/lsy5920/tangxin-zhizhe-extension/main/update.json`。
3. 远程 `version` 大于本地 `manifest.json` 的 `version` 时会提醒；版本相同但 `build` 不同时也会提醒。
4. 同一个 `version + build + changelog[0].id` 已经提醒过后不会重复弹窗，可通过插件的「清除数据缓存」重新验证。
5. GitHub raw 访问失败时不会弹窗，只会在左上角流程提示中显示更新检查失败。

## 安全与隐私

- 不要把账号密码、二维码凭证、token、deviceId 写入插件源码。
- 不要把 Supabase `service_role` 写入插件源码。
- 不要提交 `.env`、`.dev.vars`、日志、截图、浏览器 profile 和运行证据。
- 已经暴露过的密钥应立即轮换。
- 云端账号凭据应由 Worker 加密保存，插件只展示脱敏摘要。
